# 修理進捗管理システム 処理フロー仕様書

## 1. システム概要
本システムは、Google Sheetsをバックエンドデータベースとして利用した、修理案件の進捗管理および担当者割り当てを行うアプリケーションです。
Next.js (App Router) 上で動作し、Google Sheets APIを通じてデータの参照・更新を行います。

## 2. データフロー概要

```mermaid
graph TD
    User[管理者] -->|アクセス| WebApp[Next.js アプリケーション]
    WebApp -->|読み取り/書き込み| GAS[Google Sheets API]
    GAS -->|データ保存| Sheet1[Google Sheets (案件管理DB)]
    
    subgraph Google Sheets
    Sheet1 -->|シート: cases| Cases[案件データ]
    Sheet1 -->|シート: technicians| Techs[担当者データ]
    end
```

## 3. 主要プロセス

### 3.1 案件データ取得プロセス
1.  **トリガー**: 管理者が管理画面 (`/admin`) にアクセス。
2.  **処理**:
    *   `lib/googleSheets.ts` の `getCasesData()` が実行される。
    *   環境変数 `GOOGLE_SHEET_ID_CASES` で指定されたスプレッドシートの `cases` シートから全データを取得。
    *   ステータスが「完了」系のものを除外フィルター（Server Component側）。
3.  **表示**: 案件一覧として管理ダッシュボードに表示。

### 3.2 担当者データ取得プロセス
1.  **トリガー**: 管理画面のロード時、または割り当てモーダル表示時。
2.  **処理**:
    *   `lib/googleSheets.ts` の `getTechniciansData()` が実行される。
    *   `cases` と同じスプレッドシート内の `technicians` シートから担当者リストを取得。
    *   各担当者の「現在の対応件数」も合わせて取得。
3.  **利用**: 担当者割り当てドロップダウンに表示。

### 3.3 担当者割り当てプロセス
1.  **トリガー**: 管理者が案件に対して「担当者を割り当て」を実行。
2.  **処理**:
    *   Server Action (`app/actions.ts` 等) が呼び出される。
    *   Google Sheets API を使用して、対象の `cases` シートの該当行（Q列: 作業担当者）を更新。
    *   同時にステータス（P列）を「依頼中」に更新する。
3.  **結果**: 画面が再レンダリングされ、担当者とステータスが反映される。

## 4. データ構造 (Google Sheets)

### 4.1 Cases シート (案件)
| カラム | 項目名 | 説明 |
| :--- | :--- | :--- |
| A | 受付日時 | 依頼を受け付けた日時 |
| B | 送信元 | メール等の送信元 |
| C | 件名 | 依頼メール等の件名 |
| D | 店舗番号 | 店舗管理番号 |
| E | 店舗名 | 対象店舗名 |
| F | 店舗担当者 | 店舗側の担当者名 |
| G | 電話番号 | 店舗連絡先 |
| H | 住所 | 訪問先住所 |
| I | 対象機器 | 修理対象の機器 |
| J | 症状 | 故障内容 |
| K | 管理メモ | 管理者用コメント |
| L | 型式 | 型番等の情報 |
| M | 販売日 | 販売日 |
| N | 履歴 | 対応履歴 |
| O | 添付リンク | Google Drive 等の添付リンク |
| P | ステータス | 未対応/電話確認中/依頼中/対応中/修理完了/口頭完了 |
| Q | 作業担当者 | 割り当て済み担当者 |
| R | 訪問予定日時 | 担当者が登録した訪問予定日時 |

### 4.2 Technicians シート (担当者)
| カラム | 項目名 | 説明 |
| :--- | :--- | :--- |
| A | 対応エリア | 担当可能エリア |
| B | 会社名 | 所属会社名 |
| C | 担当者名 | 担当者名 |
| D | 洗剤対応 | 洗剤作業の可否 |
| E | 洗浄機対応 | 洗浄機作業の可否 |
| F | 連絡先 | 電話番号/FAX/LINE 等 |
| G | メールアドレス | 連絡先メールアドレス |

## 5. エラーハンドリング
*   **API接続エラー**: 環境変数の設定ミスや認証エラーの場合、システムは公開デモ用のテストデータを返却し、画面がクラッシュすることを防ぐフォールバック機能を持つ。
*   **シート不在エラー**: 必要なシート（`cases` や `technicians`）が存在しない場合、ログにエラーを出力する。
